# Project Mandates (GEMINI.md)

This file contains foundational instructions and project-specific standards for Gemini CLI. The instructions here take absolute precedence over general system defaults.

## 🎯 Project Overview
- **Project Name:** Advact (Advanced React Training Dashboard)
- **Goal:** A structured, high-quality learning platform for advanced React concepts.
- **Key Stack:** React 19, TypeScript, Vite, Tailwind CSS 4.

## 🛠️ Coding Standards

### TypeScript & React
- **Functional Components:** Use `export default function ComponentName() {}` for main components.
- **Typing:** Strict typing is mandatory. Avoid `any`. Prefer interfaces over types for public-facing contracts.
- **Props:** Destructure props directly in the function signature.

### Styling
- **Tailwind CSS 4:** Use utility classes for styling. Avoid custom CSS files unless absolutely necessary (e.g., global resets or complex animations).
- **Design Consistency:** Follow the existing "dark/slate" aesthetic (e.g., `bg-bg`, `bg-surface2`, `text-muted2`).

### Components
- **Granularity:** Keep components small and focused. Extract UI logic into custom hooks if it exceeds 10-15 lines.
- **UI Blocks:** Always use the established UI primitives (`Card`, `CodeBlock`, `TipBlock`, etc.) for chapter content.

## 📜 Workflow Mandates
- **Validation:** Always run `npm run lint` and `tsc` before finishing a task to ensure type safety.
- **Documentation:** When adding a new chapter section, ensure it is registered in `src/data/navigation.ts`.
- **Mock API:** If a feature requires data, use or extend the `mock-api/db.json` structure.

## 🚫 Constraints (Don'ts)
- Do not use class-based components.
- Do not introduce third-party styling libraries other than Tailwind.
- Do not modify `App.tsx` layout structure without consulting the user.
